Dual-track analysis

Keep the math clean.
Use LLMs only where they belong.

Two tracks run in parallel and never write to the same ledger. One is auditable mathematical fact. The other is AI-powered prediction of future market moves.

Track 1 / Audit

Deterministic ledger

Built on the actual transaction data, imported macro weather, and standard stress-testing instruments. Every output traces to a formula. The LLM does not touch this track.

Privacy & Safety

Privacy is not a claim.
It is a set of specific technical decisions.

CONTROL 01
Local computation

All deterministic math and cost basket analysis runs on-device. No raw financial data crosses any boundary.

CONTROL 02
Jurisdictional alignment

Deployment customised to each client's regulatory framework. Støtte adapts to local governance requirements.

AI MODEL ARCHITECTURE — ZERO DATA RETENTION
External Frontier model ZDR · signal layer only · one-way to device

Any cloud inference operates under ZDR agreements — data only comes in and not out — not stored or used for training. An external frontier model handles the predictive signal layer, processing only public market data and passing a probability vector one-way to the local model. All cost basket mapping and narrative generation runs on a captive, on-device LLM — fully local, with no external trace or data transfer.

SECURITY VERIFICATION
Independent security audit

Third-party security services verify that no sensitive data can egress the local environment — auditing all LLM-assisted code paths and API boundaries for potential data-leaking vulnerabilities before deployment.
