Dual-track analysis

Keep the math clean.
Use LLMs only where they belong.

Two tracks run in parallel and never write to the same ledger. One is auditable mathematical fact. The other is AI-powered prediction of future market moves.

Track 1 / Audit

Deterministic ledger

Solid-line output. Based on the client's actual transaction data, imported macro weather (same sources banks use: BLS, EIA, CME, Fed), and standard stress-testing instruments. Produces CCI, Monte Carlo runway scenarios, and projected cost increases for the next reporting period — all computed locally. Every output traces to a formula. The LLM does not touch this track.

SOLID LINE — MATHEMATICAL TRUTH · AUDITABLE

Track 2 / AI-powered market moves prediction

Alternative alpha layer

Dashed-line output. This layer is currently in research and testing mode. Here AI analyses alternative sources of information and produces its prediction of future macro factor moves. It is not yet integrated into the engine's calculations or hedging strategy. It does not modify the deterministic ledger. Potentially, it may be applied to sharpen scenario probabilities and runway projections.

DASHED LINE — PREDICTIVE ESTIMATE · IN TESTING

LAYER 1 — MACRO WEATHER BROADCAST

One-way feed / no inbound channel

Standardised macro vectors pushed to client devices. The broadcast server has no inbound API endpoint — it cannot receive client data. The device pulls; the server never requests. Same data sources used by institutional treasury teams.

LAYER 2 — LLM INFERENCE / HARDENED OUTBOUND

ZDR inference / four controls

Receives the CCI and stress-test results from Layer 0. Generates plain-language narrative explaining what moved and why, and outputs standard hedging category guidance. The only outbound data channel in the system. Four controls harden it.

Zero Data Retention · ZDR
Control 1: vector bucketing (5% rounding)
Control 2: ZDR provider agreement
Control 3: no covenant terms transmitted
Control 4: QMRE proxy · salted hash only
